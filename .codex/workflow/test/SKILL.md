# test

Purpose
-------
Identify or generate the minimum validation needed for the current slice.

Procedure
---------
1. Map the active issue or diff to required behaviors.
2. List nominal, edge, failure, and regression checks.
3. Choose the right test layer or review-based validation form.
4. Decide whether validation stays in `test` or needs a `debug` lane first.
5. Recommend the minimum checks needed before delivery.
